# Contributing

This project is in active development and is currently maintained by the original author.

## Contribution Philosophy

Obsidian Atomizer is designed as a deterministic, audit-safe knowledge processing system.

Contributions must preserve:
- Deterministic behavior
- Reproducibility
- Auditability

## Guidelines

- One feature or fix per pull request
- Do not introduce randomness or non-deterministic logic
- Maintain clear input/output traceability
- Include tests when applicable
- Update documentation when modifying user-facing behavior

## Areas of Interest

- New input parsers (Slack, Discord, Notion exports)
- Reliability improvements (error handling, retries)
- Performance optimizations
- Token efficiency improvements

## Before Submitting

If your change affects core pipeline behavior, open an issue first to discuss.
